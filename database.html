Database Integration Plan

Players/Coaches/Sport Professionals:

The database will store information about players, coaches, and other sport professionals, including their profiles, achievements, and contact details. It will facilitate efficient data management and retrieval for all users.

Database Structure:

Table Name Description
Users Stores information about the users including players, coaches, and sports professionals.
Sports Contains details about various sports categories available on the platform.
Articles Holds articles and guides related to outdoor sports.
TrainingPrograms Stores training routines and fitness programs for athletes.
UserInteractions Logs user activities and interactions within the platform.

Key Fields:

Field Type Description
UserID INT Primary key for the Users table.
Name VARCHAR(100) Name of the user.
Email VARCHAR(100) Email address of the user.
Role VARCHAR(50) User's role (e.g., player, coach).
SportID INT Foreign key linking to the Sports table.
ArticleID INT Primary key for the Articles table.
Title VARCHAR(200) Title of the article.
Content TEXT Content of the article.
Author VARCHAR(100) Author of the article.
Date DATE Publication date of the article.
ProgramID INT Primary key for the TrainingPrograms table.
ProgramName VARCHAR(100) Name of the training program.
Description TEXT Description of the training program.
Duration INT Duration of the training program in days.
InteractionID INT Primary key for the UserInteractions table.
UserID INT Foreign key linking to the Users table.
Action VARCHAR(100) Type of user action (e.g., login, comment).
Timestamp TIMESTAMP Time when the action occurred.
